Exemplos de Código Acessível

1. HTML Semântico Correto

Usar elementos HTML com significado correto ajuda leitores de tela e motores de busca.

Bom exemplo:

<header> <nav aria-label="Menu principal"> <ul> <li><a href="#inicio">Início</a></li> <li><a href="#sobre">Sobre</a></li> </ul> </nav> </header> <main> <article> <h1>Título do Artigo</h1> <p>Conteúdo do artigo...</p> </article> </main> <footer> <p>Copyright 2023</p> </footer>

Mau exemplo:

<div id="header"> <div class="menu"> <div><span>Início</span></div> <div><span>Sobre</span></div> </div> </div> <div class="content"> <div> <b>Título do Artigo</b> <span>Conteúdo do artigo...</span> </div> </div> <div id="footer"> <span>Copyright 2023</span> </div>

2. Imagens com Texto Alternativo

O atributo alt descreve o conteúdo da imagem para usuários que não podem vê-la.

Bom exemplo:

<img src="grafico.png" alt="Gráfico mostrando crescimento de 25% nas vendas no último trimestre" width="400" height="300">

Mau exemplo:

<img src="grafico.png" alt="gráfico"> <!-- OU pior: --> <img src="grafico.png">

3. Formulários Acessíveis

Labels associados corretamente, mensagens de erro claras e instruções.

Bom exemplo:

<form> <div> <label for="nome">Nome completo:</label> <input type="text" id="nome" name="nome" aria-required="true" aria-describedby="nome-help"> <div id="nome-help" class="help-text">Digite seu nome completo</div> </div> <fieldset> <legend>Tipo de conta</legend> <input type="radio" id="pessoa-fisica" name="tipo-conta" value="pf"> <label for="pessoa-fisica">Pessoa Física</label><br> <input type="radio" id="pessoa-juridica" name="tipo-conta" value="pj"> <label for="pessoa-juridica">Pessoa Jurídica</label> </fieldset> <button type="submit">Enviar</button> </form>

Mau exemplo:

<form> <div> <span>Nome:</span> <input type="text"> </div> <div> <span>Tipo de conta:</span> <input type="radio"> Pessoa Física <input type="radio"> Pessoa Jurídica </div> <input type="submit" value="OK"> </form>

4. Navegação por Teclado

Garantir que todos os elementos interativos sejam focáveis e tenham indicadores de foco visíveis.

Bom exemplo:

<style> /* Indicador de foco visível */ button:focus, a:focus, input:focus { outline: 3px solid #4A90E2; outline-offset: 2px; } /* Para usuários que preferem redução de movimento */ @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } } </style> <button onclick="alert('Clicado!')" aria-label="Fechar janela de diálogo"> X </button> <a href="#conteudo" onclick="skipToContent()" class="skip-link"> Pular para o conteúdo principal </a>

5. Uso de ARIA

ARIA deve ser usado quando o HTML semântico não é suficiente.

Bom exemplo:

<!-- Indicador de carregamento --> <div id="loading" aria-live="polite" aria-busy="true"> Carregando conteúdo... </div> <!-- Menu dropdown --> <button id="menu-button" aria-expanded="false" aria-haspopup="true" aria-controls="menu-dropdown"> Menu </button> <ul id="menu-dropdown" role="menu" aria-labelledby="menu-button" hidden> <li role="none"><a role="menuitem" href="#item1">Item 1</a></li> <li role="none"><a role="menuitem" href="#item2">Item 2</a></li> </ul> <!-- Mensagem de alerta --> <div role="alert" aria-live="assertive" id="error-message"> Erro: Preencha todos os campos obrigatórios. </div>
